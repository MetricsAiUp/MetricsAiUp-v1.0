# Zone Occupancy Classifier — Training Bundle

Drop-in replacement for Claude Vision in the zone-mapper pipeline.
Trains a convnet to predict **occupied / free** for each zone crop,
with an auxiliary head for **vehicle body type** (10 classes).

## Why this stack

| Requirement                | Decision                                                            |
|----------------------------|---------------------------------------------------------------------|
| Maximum accuracy           | **ConvNeXt-V2 Large** (ImageNet-22k→1k, 198M params, 87.3% IN1k)   |
| Training on RTX 3070 (8GB) | fp16 autocast + batch=8 @ 384px + gradient clipping                |
| Inference on RTX 3070      | fp16 weights, ~30-50 ms / image @ 384px                            |
| Clean, clean labels        | Use only Claude `confidence=HIGH`+`MEDIUM`, drop test zones        |
| Strong imbalance per zone  | **WeightedRandomSampler** — balances occupied/free in each batch   |
| Generalization to "future" | Chronological train/val/test split (80/10/10 by timestamp)         |
| Body type recycling        | Auxiliary cross-entropy head, masked loss on unknown samples       |

The alternative `configs/eva02_large.yaml` gives the absolute max accuracy
(~90% IN1k, state of the art 2024-2025) but needs 24GB+ VRAM for training.
Use it on a rented A100/H100 if you want the last 1-2% of accuracy.

```

├── dataset/                  # hardlinked to original data/dataset/
│   ├── images/               # 34,913 JPEG crops
│   └── labels/               # 34,913 Claude-annotated JSONs
├── splits/                   # generated by prepare_splits.py
│   ├── train.jsonl           # ~25k samples, 15–21 apr
│   ├── val.jsonl             # ~3k samples, 22 apr
│   ├── test.jsonl            # ~3k samples, 23 apr
│   ├── body_classes.json     # 10 canonical body types
│   └── zone_stats.json       # per-zone counts per split
├── scripts/
│   ├── prepare_splits.py     # label normalization + split creation
│   ├── dataset.py            # PyTorch Dataset + transforms + loaders
│   ├── model.py              # OccupancyModel (backbone + 2 heads)
│   ├── train.py              # main trainer (fp16, cosine LR, EMA-lite)
│   ├── evaluate.py           # test eval + threshold sweep + per-zone
│   ├── predict.py            # single image / dir / glob inference
│   └── serve.py              # FastAPI server — drop-in for Claude Vision
├── configs/
│   ├── convnextv2_large.yaml # default (recommended)
│   ├── convnextv2_base.yaml  # fallback if Large OOMs
│   └── eva02_large.yaml      # max-accuracy (A100 recommended)
├── checkpoints/              # (filled after training)
├── requirements.txt
└── README.md
```

## 5-minute quickstart

```bash
cd training_bundle/

# 1. Python env (Python 3.10+)
python -m venv .venv && source .venv/bin/activate

# 2. Install PyTorch matching your CUDA (RTX 3070 → cu121)
pip install torch==2.4.1 torchvision==0.19.1 \
    --index-url https://download.pytorch.org/whl/cu121
pip install -r requirements.txt

# 3. Generate splits (fast, runs in seconds)
python scripts/prepare_splits.py

# 4. Train. On 3070 ~7-10h for 8 epochs.
python scripts/train.py --config configs/convnextv2_large.yaml

# 5. Evaluate on test set
python scripts/evaluate.py \
    --ckpt checkpoints/convnextv2_large_384/best.pt \
    --split test \
    --out checkpoints/convnextv2_large_384/test_report.json

# 6. Start inference server
python scripts/serve.py \
    --ckpt checkpoints/convnextv2_large_384/best.pt \
    --port 8000
```

## If Large OOMs on 3070

Try in this order:

1. `configs/convnextv2_base.yaml` (batch=16 @ 384, ~4GB VRAM)
2. Same large config but `img_size: 224` and `batch_size: 16`
3. Enable gradient checkpointing (add `grad_checkpoint: true`) — not wired by
   default because timm `set_grad_checkpointing(True)` works for ConvNeXt.

## Training on a different GPU

Bundle is self-contained. On the rented A100 / 4090 / H100:

```bash
rsync -av training_bundle/ rental:/home/user/training_bundle/
ssh rental
cd training_bundle/
# edit configs/convnextv2_large.yaml → batch_size: 32, img_size: 384
# or just use eva02_large.yaml
python scripts/train.py --config configs/eva02_large.yaml
```

Then copy only `checkpoints/<run>/best.pt` (~500-800 MB) back to 3070
and run `serve.py`.

## Integration with zone-mapper backend

`serve.py` exposes the same interface Claude Vision effectively has:

```
POST http://localhost:8000/analyze
Content-Type: image/jpeg
Body: <raw jpeg bytes>

→ 200 { "occupied": true, "occupancy_prob": 0.987,
        "body": "sedan", "body_prob": 0.91, "processing_ms": 42 }
```

To wire into `modules/zone-mapper/backend/lib/vision.js`, just add a new
function `analyzeLocal(jpegBuffer)` that POSTs to `http://localhost:8000/analyze`
and returns the same shape expected by `autoPoll.js`. Claude path can be
kept as a fallback.

## Evaluation metrics to watch

| Metric                                   | Target                         |
|------------------------------------------|--------------------------------|
| `val_f1` / `test_f1` (occupancy)         | > 0.97                         |
| `test_acc` overall                       | > 0.97                         |
| Worst-zone `f1` (Своб. 04 / 07 / 05)     | > 0.90 — imbalance-sensitive   |
| `body_acc` on labeled samples            | > 0.85                         |
| Inference latency @ 384px fp16 on 3070   | < 50 ms single, < 8 ms batched |

If worst-zone F1 is low even with balanced sampling, collect more "free"
frames from those zones (Ford Transit parked permanently issue) before
retraining.

## Retraining on fresh data

Once AutoPoll resumes and more days accumulate:

```bash
# Rerun prepare_splits — it picks up everything in dataset/labels/
python scripts/prepare_splits.py

# Continue from best.pt (optional)
python scripts/train.py \
    --config configs/convnextv2_large.yaml \
    --resume checkpoints/convnextv2_large_384/best.pt
```
