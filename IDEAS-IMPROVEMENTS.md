# Идеи улучшений MetricsAiUp

> Сгенерировано: 2026-04-07
> Для оценки и выбора Артёмом

---

## ВЫСОКИЙ ПРИОРИТЕТ

### 1. Валидация входных данных на бэкенде
- Сейчас POST/PUT маршруты принимают `req.body` напрямую без проверки
- Нужно: Zod или Joi схемы для каждого эндпоинта
- Файлы: `backend/src/routes/zones.js`, `posts.js`, `workOrders.js`, `users.js`

### 2. Drag-and-drop планировщик заказ-нарядов
- Таймлайн DashboardPosts сейчас read-only
- Идея: перетаскивание ЗН на посты/время, визуальное планирование смены
- Файл: новая страница или расширение DashboardPosts

### 3. Двусторонняя синхронизация с 1С Альфа-Авто
- Сейчас: импорт Excel вручную, данные только читаются
- Идея: Push завершённых ЗН обратно в 1С через API, автоматический импорт
- Файлы: `backend/src/routes/workOrders.js`, `Data1C.jsx`

### 4. Экспорт аналитики в Excel/PDF
- Графики и таблицы Analytics нельзя экспортировать
- Идея: кнопка «Экспорт» → xlsx/pdf отчёт с графиками
- Файл: `Analytics.jsx`, использовать xlsx (уже в проекте)

### 5. Управление сменами
- Есть настройки начало/конец смены в DashboardPosts, но нет полноценного управления
- Идея: создание смен, привязка работников к сменам, передача смен
- Новая страница `/shifts`

---

## СРЕДНИЙ ПРИОРИТЕТ

### 6. Мобильная адаптивность
- Sidebar фиксированный 160px — неюзабельный на телефоне
- Идея: hamburger-меню на экранах < 768px, responsive таблицы
- Файлы: `Layout.jsx`, `Sidebar.jsx`, все страницы с таблицами

### 7. Уведомления о критических событиях
- Сейчас: polling каждые 5 сек, тихое обновление
- Идея: Socket.IO push для критических событий → toast + звук
  - Клиент не приехал (no_show)
  - Авто без работы > 15 мин (vehicle_idle)
  - Пост простаивает > 30 мин (post_free)
- Файлы: `Dashboard.jsx`, `hooks/useSocket.js`

### 8. Переключатель слоёв на карте
- Viewer показывает все элементы одновременно
- Идея: чекбоксы вкл/выкл: посты, зоны, камеры, проезды, подписи
- Файл: `MapViewer.jsx`

### 9. Тепловая карта загрузки постов
- Analytics показывает графики, но нет визуализации по времени дня
- Идея: heatmap (ось X — часы дня, ось Y — посты, цвет — загрузка)
- Файл: `Analytics.jsx` или новый таб

### 10. Экспорт карты в PNG/PDF
- MapViewer: нет возможности скачать карту
- Идея: кнопка «Скачать карту» → Konva stage → toDataURL → скачивание
- Файл: `MapViewer.jsx`

### 11. Аудит-лог действий пользователей
- Сейчас: нет записи кто что менял
- Идея: middleware для логирования мутаций (кто, что, когда, старое/новое значение)
- Файлы: новая таблица `AuditLog` в Prisma, middleware

### 12. Snap-to-grid в редакторе карты
- Элементы свободно перемещаются
- Идея: привязка к сетке при перетаскивании (snap каждые 10/20px)
- Файл: `MapEditor.jsx`

### 13. Undo/Redo в редакторе карты
- Нет возможности отменить действие
- Идея: стек истории (Ctrl+Z / Ctrl+Shift+Z)
- Файл: `MapEditor.jsx`

### 14. Дубликация элементов в редакторе
- Нет копирования элемента
- Идея: Ctrl+D или кнопка «Дублировать» в панели свойств
- Файл: `MapEditor.jsx`

### 15. Легенда на карте просмотра
- Нет объяснения цветов и символов
- Идея: компактная легенда в углу (статусы постов, камеры, зоны)
- Файл: `MapViewer.jsx`

## НИЗКИЙ ПРИОРИТЕТ

### 16. Предиктивная аналитика (ML)
- Прогноз когда пост освободится (на основе исторических данных)
- Прогноз загрузки на завтра/неделю
- Рекомендации по оптимальному расписанию

### 17. Мультитенантность (несколько СТО)
- Сейчас: одна СТО на инстанс
- Идея: location_id для всех данных, переключатель СТО в header
- Масштабная доработка архитектуры

### 18. PWA + Push-уведомления
- Установка как приложение на телефон
- Push-уведомления для мастеров о новых ЗН

### 19. Таймер на посту
- Обратный отсчёт до плановой сдачи ЗН прямо на карте
- Цвет меняется: зелёный → жёлтый → красный по мере приближения дедлайна
- Файлы: `MapViewer.jsx`, `STOMap.jsx`

### 20. Фото/видео фиксация
- Мастер делает фото повреждений при приёмке через камеру телефона
- Привязка к ЗН и сессии авто
- Хранение в бэкенде (S3 или локально)

### 21. QR-коды для автомобилей
- Генерация QR при въезде → механик сканирует для открытия ЗН
- Файл: новый компонент `QRGenerator.jsx`

### 22. Дашборд для мобильного механика
- Упрощённый вид: только свой пост, текущий ЗН, таймер
- Файл: новая страница `/my-post`

### 23. Интеграция с мессенджерами
- Отправка уведомлений в Telegram при критических событиях
- Бот для проверки статуса поста

### 24. Тёмная/светлая тема для карты
- Карта MapViewer сейчас одинаковая в обеих темах
- Идея: инверсия цветов фона при тёмной теме

### 25. Версионирование карт
- Сохранять историю изменений layout карты
- Возможность откатиться к предыдущей версии
- Файлы: `MapLayout` модель + API
