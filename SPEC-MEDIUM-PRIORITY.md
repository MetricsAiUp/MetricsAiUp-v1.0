## 6. Мобильная адаптивность
**Оценка:** 4-5 часов | **Сложность:** Средняя

### Проблема
Sidebar фиксирован 160px — на телефоне занимает половину экрана. Таблицы не помещаются, модальные окна выходят за экран. Мастера на СТО часто смотрят с планшетов — сейчас неюзабельно.

### Что сделать
- **Sidebar:** `< 768px` → скрыт, hamburger-кнопка в header. Выезжает слева overlay. `≥ 768px` — как сейчас
- **Header:** компактнее на мобильных, имя пользователя скрыть (только аватар)
- **Таблицы:** горизонтальный скролл, sticky первая колонка (номер/имя)
- **Модалки:** `max-width: 90vw`, `max-height: 85vh`, скролл внутри
- **Dashboard карточки:** 1 колонка на мобильных, 2 на планшетах, 4 на десктопе
- **Карта:** touch-события (pinch zoom, swipe pan)

### Файлы
`Layout.jsx`, `Sidebar.jsx`, все страницы с таблицами, `MapViewer.jsx`

## 7. Уведомления о критических событиях
**Оценка:** 4-5 часов | **Сложность:** Средняя

### Проблема
Критические ситуации (авто без работы, клиент не приехал) обнаруживаются только при ручном просмотре дашборда. Мастер может не заметить — теряется время и деньги.

### Что сделать
- **Socket.IO → Toast:** при получении события `recommendation` → показать toast с иконкой и звуком
- **Типы уведомлений:**
  - 🔴 `no_show` — "Клиент Х не приехал на ЗН Y" (звук)
  - 🟡 `vehicle_idle` — "Авто АВ123 без работы 15 мин на посту 3"
  - 🔵 `post_free` — "Пост 7 свободен более 30 мин"
  - 🟠 `work_overtime` — "ЗН 12345 превышает нормо-часы на 20%"
- **Центр уведомлений:** иконка 🔔 в header с бейджем непрочитанных → dropdown список
- **Настройки:** какие типы показывать (профиль пользователя)
- **Звук:** опциональный, по умолчанию только для `no_show` и `overtime`

### Файлы
`Layout.jsx` (header), `hooks/useSocket.js`, новый `NotificationCenter.jsx`

## 8. Переключатель слоёв на карте
**Оценка:** 1-2 часа | **Сложность:** Простая

### Проблема
MapViewer показывает все элементы одновременно. При 10 постах, 15 камерах, 7 зонах + стены + проезды — карта перегружена. Нельзя посмотреть только камеры или только посты.

### Что сделать
- Панель в правом верхнем углу карты:
  ```
  Слои:
  ☑ Посты          ☑ Камеры (FOV)
  ☑ Зоны           ☑ Проезды
  ☑ Стены/двери    ☑ Подписи
  ☑ Фон (чертёж)   ☑ Здание
  ```
- Чекбокс вкл/выкл каждый тип элемента
- Состояние сохраняется в localStorage
- Сворачиваемая панель (кнопка ▾ / ▴)

### Файлы
`MapViewer.jsx` — добавить state `visibleLayers`, фильтр при рендере элементов

## 9. Тепловая карта загрузки постов
**Оценка:** 3-4 часа | **Сложность:** Средняя

### Проблема
Analytics показывает общую загрузку за период, но не видно в какие часы какие посты перегружены. Мастер не может оптимизировать расписание — не знает "узкие места" дня.

### Что сделать
- **Heatmap-компонент** (таблица-сетка):
  ```
           08:00  08:30  09:00  09:30  10:00  ...  19:30
  Пост 1   ■■■    ■■■    ■■■    ■■■    ■■■         ■■■
  Пост 2   ■■■    □□□    ■■■    ■■■    □□□         □□□
  Пост 3   □□□    □□□    ■■■    ■■■    ■■■         ■■■
  ...
  ```
  - Зелёный = свободен, Жёлтый = занят, Красный = перегружен
- **Tooltip:** "Пост 3, 10:00-10:30 — загрузка 85%, 2 ЗН"
- **Фильтры:** по дню недели, конкретной дате, периоду
- **Данные:** из `analytics-history.json` (hourly breakdown уже есть)

### Файлы
Новый компонент `Heatmap.jsx`, добавить в `Analytics.jsx` как новый блок

## 10. Экспорт карты в PNG/PDF
**Оценка:** 1 час | **Сложность:** Простая

### Проблема
MapViewer нельзя скачать для печати. Мастер хочет распечатать карту с текущими статусами постов и повесить на стену.

### Что сделать
- Кнопка "📥 Скачать" в toolbar MapViewer
- Konva `stage.toDataURL({ pixelRatio: 2 })` → скачивание PNG
- Опция PDF: canvas → jsPDF (A4 landscape) с заголовком "Карта СТО на {дата} {время}"
- Чекбокс "Включить фон (чертёж)" — без фона элементы на белом/тёмном фоне

### Файлы
`MapViewer.jsx` — кнопка + логика экспорта

## 11. Аудит-лог действий пользователей
**Оценка:** 3-4 часа | **Сложность:** Средняя

### Проблема
Нет записи кто что менял. Удалили пользователя — не найти кто. Изменили карту — не откатить. Для продукта нужна подотчётность.

### Что сделать
- **Prisma модель:**
  ```
  AuditLog { id, userId, userName, action, entity, entityId,
             oldData (JSON), newData (JSON), ip, createdAt }
  ```
- **Middleware:** перехватывает POST/PUT/DELETE, записывает в лог автоматически
- **Страница `/audit`** (только admin):
  - Таблица: время, пользователь, действие, объект
  - Фильтры: по пользователю, типу действия, дате
  - Детали: клик → показать старые/новые данные (diff)
- **Actions:** create_user, update_zone, delete_post, save_map_layout, acknowledge_recommendation...

### Файлы
Новая модель Prisma, новый middleware, новая страница, новый роут

## 12. Snap-to-grid в редакторе карты
**Оценка:** 1-2 часа | **Сложность:** Простая

### Проблема
Элементы перемещаются пиксель-в-пиксель — трудно выровнять. 10 постов разного размера выглядят криво.

### Что сделать
- При перемещении элемент округляется до ближайшей линии сетки (каждые 10px)
- При resize — snap размеров тоже к 10px
- Кнопка "🧲 Snap" рядом с "Сетка" (вкл/выкл)
- **Alignment guides:** тонкие синие линии появляются когда элемент выровнен по центру или краю другого элемента

### Файлы
`MapEditor.jsx` — обработчики `onDragEnd`, `onTransformEnd`

## 13. Undo/Redo в редакторе карты
**Оценка:** 2 часа | **Сложность:** Простая

### Проблема
Случайно удалил элемент или переместил не туда — нельзя отменить. Приходится перезагружать.

### Что сделать
- Стек истории: массив JSON-снимков `elements[]` (до 50 шагов)
- **Ctrl+Z** — отмена → восстановить предыдущий snapshot
- **Ctrl+Shift+Z** — повтор → восстановить следующий
- Кнопки ↩ ↪ в toolbar
- Записываются: добавление, удаление, перемещение, resize, изменение свойств

### Файлы
`MapEditor.jsx` — `useRef` для стека, `useEffect` для keyboard shortcuts

### Проблема

## 15. Легенда на карте просмотра
**Оценка:** 1 час | **Сложность:** Простая

### Проблема
Пользователь не знает что означают цвета. Зелёный — это свободный пост или зона? Красный кружок — проблема или камера?

### Что сделать
- Компактная легенда в правом нижнем углу:
  ```
  ┌─────────────────────┐
  │ 🟢 Свободен         │
  │ 🟡 Занят            │
  │ 🔴 Простой           │
  │ 🟣 В работе          │
  │ ─ ─ Проезд           │
  │ 🔴○ Камера           │
  │ ▭ Зона              │
  └─────────────────────┘
  ```
- Сворачиваемая (кнопка ▾ / ▴)
- Адаптивная: на мобильных — иконки без текста

### Файлы
Новый `MapLegend.jsx`, подключить в `MapViewer.jsx`
