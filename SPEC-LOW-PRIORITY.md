## 16. Предиктивная аналитика (ML)
**Оценка:** 20+ часов | **Сложность:** Сложная

### Идея
На основе исторических данных (30+ дней) прогнозировать:
- **Когда пост освободится** — на карте показывать "≈ свободен через 45 мин"
- **Загрузку на завтра/неделю** — график прогноза по часам
- **Оптимальное расписание ЗН** — алгоритм минимизации простоев
- **Вероятность no-show** — для конкретных клиентов по истории

### Реализация
- Python backend (Flask/FastAPI) + scikit-learn / Prophet
- Модель: RandomForest или LSTM для временных рядов
- Данные: из Prisma (postStays, workOrders, sessions за 30+ дней)
- REST API: `GET /api/predictions/post/:id/free-at` → `{ estimatedFreeAt, confidence }`
- Фронтенд: виджет прогноза на карте и в PostsDetail

## 17. Мультитенантность (несколько СТО)
**Оценка:** 15-20 часов | **Сложность:** Сложная

### Идея
Один инстанс для сети СТО (франшиза ПарадАвто):
- `locationId` во всех моделях (User, Post, Zone, Session, WorkOrder, MapLayout)
- Переключатель СТО в header: dropdown с названиями филиалов
- Общий дашборд для директора: сводка по всем СТО на одном экране
- Каждая СТО — своя карта, свои посты, свои камеры
- Пользователь может иметь доступ к 1+ СТО

### Архитектура
```
Модель Location { id, name, address, timezone }
Все запросы фильтруются по текущему locationId
Middleware: req.locationId из JWT или header
```

## 18. PWA + Push-уведомления
**Оценка:** 4-5 часов | **Сложность:** Средняя

### Идея
Установка как приложение на телефон:
- `manifest.json` — иконка, название, цвет темы
- **Service Worker** — кеширование shell (HTML/CSS/JS), offline fallback
- **Push-уведомления** через Web Push API:
  - Новый ЗН назначен мастеру
  - Клиент не приехал (no_show)
  - Пост простаивает 30+ мин
- Подписка: бэкенд хранит push subscription, рассылает через web-push npm

### Выгода
Мастер на СТО ходит по цеху с телефоном — получает push без открытия браузера

## 19. Таймер обратного отсчёта на посту
**Оценка:** 2-3 часа | **Сложность:** Простая

### Идея
На карте и в DashboardPosts — живой обратный отсчёт до плановой сдачи ЗН:
```
Пост 3: ЗН 12345
🟢 ⏱ 02:15:30 (до сдачи)

Пост 7: ЗН 67890
🟡 ⏱ 00:12:45 (скоро дедлайн!)

Пост 5: ЗН 11111
🔴 ⏱ -00:23:10 (просрочено!)
```
- **Зелёный:** > 30 мин до дедлайна
- **Жёлтый:** 10-30 мин
- **Красный:** < 10 мин
- **Мигающий красный:** просрочено

### Данные
`estimatedEnd` из `dashboardData.posts[].timeline[].estimatedEnd`

## 20. Фото/видео фиксация при приёмке
**Оценка:** 5-6 часов | **Сложность:** Средняя

### Идея
Мастер фотографирует повреждения авто при приёмке:
- Кнопка "📷 Фото" в карточке сессии/ЗН
- `getUserMedia()` → камера телефона → снимок
- Или загрузка готовых фото из галереи
- Привязка к vehicleSessionId + workOrderId
- Галерея в карточке сессии: миниатюры → полноэкранный просмотр
- Хранение: `/data/photos/{sessionId}/` или S3
- Бэкенд: `POST /api/sessions/:id/photos` (multipart/form-data)

### Идея

```
QR содержит: https://artisom.dev.metricsavto.com/#/session/{id}
```

- Компонент `QRBadge.jsx`: отображает QR + номер авто + время

**Оценка:** 4-5 часов | **Сложность:** Средняя

## 23. Интеграция с Telegram
**Оценка:** 3-4 часа | **Сложность:** Средняя

### Идея
Telegram-бот для мастеров и директора:

**Уведомления (push):**
- "🔴 Клиент АВ123 не приехал на ЗН 12345 (запись 10:00)"
- "🟡 Пост 7 свободен уже 45 мин"
- "⚠️ ЗН 67890 превышает нормо-часы: 3.2ч / 2.5ч"

**Команды:**
- `/status` — "Загрузка СТО: 70%, 7/10 постов заняты"
- `/post 3` — "Пост 3: OPEL CROSSLAND, ЗН 12345, мех. Пузанов, осталось 1:15"
- `/free` — "Свободные посты: 2, 8, 10"
- `/report` — Сводка за день (PDF)

### Реализация
- `npm install node-telegram-bot-api`
- Бэкенд: `backend/src/services/telegramBot.js`
- Webhook: бэкенд → Telegram Bot API
- Настройка: bot token в .env, chat IDs пользователей

## 24. Тёмная/светлая тема для карты
**Оценка:** 1-2 часа | **Сложность:** Простая

### Идея
MapViewer сейчас выглядит одинаково в обеих темах:
- **Тёмная:** тёмный фон canvas (#1e293b), белые подписи, яркие элементы, фон чертежа с opacity 0.3
- **Светлая:** белый фон, тёмные подписи, обычная яркость фона
- Зоны: в тёмной теме более яркие заливки, в светлой — приглушённые
- Камеры: в тёмной — более яркий конус FOV

## 25. Версионирование карт
**Оценка:** 3-4 часа | **Сложность:** Средняя

### Идея
История изменений layout карты:
- При каждом сохранении в редакторе → `version++`
- Модель: `MapLayoutVersion { id, layoutId, version, elements, savedBy, createdAt }`
- UI в редакторе: dropdown "Версия 5 (сегодня) / Версия 4 (вчера) / ..."
- Кнопка "Откатить к версии N"
- Сравнение: "Добавлено 2 камеры, перемещён пост 3"
