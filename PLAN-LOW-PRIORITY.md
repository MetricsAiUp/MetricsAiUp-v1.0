## #16. Предиктивная аналитика (ML)

### Этапы
| # | Задача | Время |
|---|--------|-------|
| 16.1 | Сбор и очистка данных: экспорт postStays, workOrders за 30+ дней из Prisma в CSV | 2ч |
| 16.2 | Исследование: Jupyter notebook, анализ паттернов (загрузка по дням/часам, duration distribution) | 4ч |
| 16.3 | Модель прогноза загрузки: Prophet или RandomForest (по часам на 7 дней вперёд) | 4ч |
| 16.4 | Модель прогноза длительности ЗН: по типу работ, марке авто, механику | 3ч |
| 16.5 | Python API (FastAPI): `GET /predict/load?date=&post=` → `{ hourly: [...] }` | 2ч |
| 16.6 | Интеграция: виджет прогноза в Analytics + на карте (прогноз свободных постов) | 3ч |
| 16.7 | Автообновление модели: retrain каждую ночь по cron | 2ч |

### Зависимости
- Нужно минимум 30 дней реальных данных
- Python 3.11 (есть в контейнере)

### Критерий готовности
- Прогноз загрузки на неделю вперёд с точностью ≥ 70%
- Прогноз "пост освободится через ~X мин" на карте
