# MetricsAiUp — Полная документация проекта

## 1. Обзор проекта

**MetricsAiUp** — система мониторинга автосервиса (СТО), отслеживающая загрузку постов, заказ-наряды, сессии автомобилей и события с камер видеонаблюдения.

- **Домен**: `dev.metricsavto.com`
- **Среда**: Docker-контейнер, Nginx на порту 8080
- **Языки**: Русский (по умолчанию), Английский
- **Тема**: Светлая / Тёмная (glassmorphism)

## 5. Карта СТО (STOMap.jsx)

### Схема расположения

```
┌─────────┬──────────────────────────────────────────┐
│Парковка │  Пост 5  │  Пост 6  │  Пост 7  │  Пост 8  │  Пост 9  │
│         ├──────────┴──────────┴──────────┴──────────┴──────────┤
│ Въезд ↔ │                    Проезд                              │
│         ├──────────┬──────────┬──────────┬──────────┬──────────┤
│Парковка │  Пост 1  │  Пост 2  │  Пост 3  │  Пост 4  │ Пост 10  │
└─────────┴──────────────────────────────────────────┘
```

### Параметры
- **Canvas**: 860×460px, масштабируется на всю ширину
- **Посты**: 130×120px каждый, зазор 10px
- **Зоны**: 6 (ремонт верх, проезд, ремонт низ, парковка верх, въезд, парковка низ)
- **Камеры**: 10шт (КАМ01-10), с FOV-линиями, кликабельные

### Статусы постов
| Статус | Цвет | Значение |
|---|---|---|
| `free` | 🟢 #10b981 | Свободен |
| `occupied` | 🟡 #f59e0b | Занят |
| `occupied_no_work` | 🔴 #ef4444 | Простой (авто без работы) |
| `active_work` | 🔵 #6366f1 | Активная работа |

### Информация на карточке поста
1. Название поста + статус
2. Марка-модель авто
3. Номер ЗН
4. Исполнитель
5. Время: установлен → прогноз снятия
6. Гос. номер (внизу)

### Интерактивность
- **Клик на пост** → модальное окно с деталями + кнопка "Перейти к посту"
- **Клик на камеру** → модальное окно стрима (placeholder для HLS)

## 6. Дашборд постов (DashboardPosts.jsx)

### Gantt-таймлайн
- **Формат**: горизонтальные полосы на временной шкале смены
- **Шкала**: часовые + 30-минутные отметки, настраиваемый период
- **Текущее время**: красная вертикальная линия
- **Блоки ЗН**: кликабельные, цветовая кодировка по статусу

### Статусы блоков
| Статус | Цвет | Описание |
|---|---|---|
| `completed` | 🟢 зелёный | Завершён |
| `in_progress` | 🔵 индиго | В работе |
| `scheduled` | ⚪ серый | Запланирован |
| `overdue` | 🔴 красный | Просрочен (текущее время > прогноз) |

### Настройки (модальное окно)
- Начало/конец смены (время)
- Количество постов (1-30)
- Режим: Demo / Live

### Свободные ЗН (живая очередь)
Таблица: номер ЗН, авто, тип работ, тип поста (легковой/грузовой), нормочасы, клиент, примечание.

## 7. Страница "Посты" (PostsDetail.jsx)

### Навигация
- В sidebar пункт "Посты" со стрелкой → раскрывается подменю с 10 постами
- Клик на пост → `/posts-detail?post=post-N`

### Содержимое страницы поста

#### Заголовок
Название поста, тип, зона, переключатель периода (Сегодня/Вчера/Неделя/Месяц)

#### Таймлайн (только для "Сегодня")
Gantt-полоса с ЗН за текущую смену (формат как на дашборде постов)

#### Сводные показатели
- **План / Факт / Макс. ёмкость** — горизонтальные полоски
- **Загрузка %** — зелёная ≥70%, жёлтая 30-70%, красная <30%
- **Эффективность %** — зелёная ≥80%, жёлтая 50-80%, красная <50%

#### Сворачиваемые секции (все изначально развёрнуты)
1. **Заказ-наряды** — список ЗН с номером, авто, типом работ, нормочасами, план/факт %
2. **Исполнители** — ФИО, роль, отработано часов, закрыто ЗН
3. **Алерты** — нарушения, задержки (если есть)
4. **Лог событий** — хронология: установка/снятие авто, начало/конец работ, простой
5. **Статистика** — работы по группам (горизонтальные полоски), авто по маркам, среднее время на ЗН
6. **Камеры** — превью камер с видом на пост
7. **Загрузка по дням** — календарная таблица за 7 дней (загрузка %, эффективность %, нормочасы)

## 8. Система авторизации

### AuthContext.jsx
- **Мок-авторизация**: статичный JWT-токен из `auth-login.json`
- **Пользователь**: Admin (admin@metricsai.up), роль admin
- **Права**: 15 permissions (view_*, manage_*)
- **Хранение**: localStorage (`token`)

### API-обёртка
```
api.get('/api/zones')           → fetch api/zones.json
api.get('/api/sessions?status=completed') → fetch api/sessions-completed.json
api.get('/api/dashboard/metrics?period=7d') → fetch api/dashboard-metrics-7d.json
api.post('/api/auth/login')     → fetch api/auth-login.json
```

### Права доступа
```
view_dashboard, view_zones, view_sessions, view_work_orders,
view_analytics, view_events, view_cameras, view_recommendations,
view_posts, manage_users, manage_roles, manage_settings,
manage_cameras, manage_work_orders, manage_zones
```

## 11. Модели данных

### Пост (Post)
```json
{
  "id": "post-1",
  "number": 1,
  "name": "Пост 1",
  "type": "light|heavy|diagnostics",
  "status": "free|occupied|active_work|occupied_no_work",
  "zone": "Ремонтная зона",
  "currentVehicle": { "plateNumber", "brand", "model", "color" },
  "timeline": [ WorkOrderBlock[] ]
}
```

### Блок ЗН (WorkOrderBlock)
```json
{
  "id": "tl-1-1",
  "workOrderNumber": "ЗН-0301",
  "plateNumber": "К555МН77",
  "brand": "Hyundai", "model": "Solaris",
  "workType": "ТО-1",
  "status": "completed|in_progress|scheduled",
  "startTime": "2026-03-30T08:00:00",
  "endTime": "2026-03-30T09:20:00",
  "estimatedEnd": "...",
  "normHours": 1.5, "actualHours": 1.3,
  "master": "Иванов А.С.", "worker": "Петров Д.В.",
  "note": "Ожидает запчастей"
}
```

### Сессия (VehicleSession)
```json
{
  "id": "uuid",
  "plateNumber": "А123ВС77",
  "entryTime": "ISO", "exitTime": null,
  "status": "active|completed",
  "trackId": "track_1",
  "zoneStays": [{ "zoneId", "entryTime", "exitTime", "zone" }],
  "postStays": [{ "postId", "startTime", "endTime", "hasWorker", "isActive", "activeTime", "idleTime" }]
}
```

### Событие (Event)
```json
{
  "id": "uuid",
  "type": "vehicle_entered_zone|post_occupied|worker_present|work_activity|...",
  "zone": { "id", "name", "type" },
  "post": { "id", "name", "status" },
  "confidence": 0.95,
  "createdAt": "ISO"
}
```

### Заказ-наряд (WorkOrder)
```json
{
  "id": "uuid",
  "orderNumber": "ЗН-2026-0301",
  "plateNumber": "А123ВС77",
  "workType": "ТО-2",
  "normHours": 3, "actualHours": 2.3,
  "status": "scheduled|in_progress|completed|no_show|cancelled",
  "scheduledTime": "ISO",
  "links": [{ "vehicleSessionId", "confidence", "matchType": "plate" }]
}
```

## 15. Текущие ограничения и плейсхолдеры

1. **Socket.IO отключён** — используется polling (5 сек)
2. **Камеры** — плейсхолдеры, нет реального стрима (готово под HLS)
3. **API** — статичные JSON-моки, нет реального бэкенда
4. **Распознавание номеров** — мок SVG-плашки
5. **Фото авто на посту** — плейсхолдер
6. **CSV/Excel импорт** — работает, но не сохраняет (мок)
7. **База данных** — Prisma настроена, но заполнена моками
8. **Авторизация** — статичный токен, нет реальной проверки
